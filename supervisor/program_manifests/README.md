# Program Manifests

Program manifests are small mutable backlog files.

They answer:
- what packet is already complete
- what packet is next
- when the active program should stop and route to human review

They do not replace:
- seed specs in `research_areas/seeds/`
- immutable genesis in `supervisor/program_genesis/`
- tactical debate logs in `research_areas/debates/`

Current policy:
- manifests are human-authored or human-approved
- the supervisor may read them for orientation
- manifests do not self-authorize new programs
